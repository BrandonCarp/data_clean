React components → PascalCase

Navbar.tsx, CsvPreview.tsx, DarkModeToggle.tsx

function CsvPreview() { ... }

Types / Interfaces / Enums → PascalCase

type OrderRow = { ... }

interface User { ... }

Functions & helpers → camelCase

parseMoney, loadCsv, formatDate

File names for helpers: usually kebab-case or camelCase:

parse-money.ts or parseMoney.ts (just be consistent inside lib/)

Variables, props, state → camelCase

const isDarkMode = ...

const [rows, setRows] = useState<Row[]>([])

Route segments (folders in app/) → kebab-case

app/user-settings/page.tsx → /user-settings

app/raw-orders/page.tsx → /raw-orders

Next.js special files (framework stuff) → all lowercase

page.tsx, layout.tsx, loading.tsx, error.tsx, not-found.tsx, route.ts

So your mental shortcut can be:

PascalCase → “things that are types or components”

camelCase → “things that do stuff or hold values”

kebab-case → “URLs / generic filenames”
